# GUI 设计规范

## 概述

本文档记录 RedC GUI 的设计规范，包括配色、边框、按钮等样式定义，供后续维护参考。

---

## 1. 页面整体风格

### 1.1 卡片容器

```html
<div class="bg-white rounded-xl border border-gray-100 p-5">
```

| 属性 | 值 |
|------|-----|
| 背景色 | `bg-white` (白色) |
| 圆角 | `rounded-xl` (12px) |
| 边框 | `border border-gray-100` (灰色-100, 1px) |
| 内边距 | `p-5` (20px) 或 `p-4` (16px) |

### 1.2 页面背景

```html
<div class="min-h-screen bg-gray-50">
```

| 属性 | 值 |
|------|-----|
| 最小高度 | `min-h-screen` (100vh) |
| 背景色 | `bg-gray-50` (灰色-50) |

## 10. 响应式断点

| 断点 | 前缀 | 宽度 |
|------|------|------|
| 手机 | `sm:` | ≥640px |
| 平板 | `md:` | ≥768px |
| 桌面 | `lg:` | ≥1024px |

**常用响应式写法：**

```html
<!-- 高度响应式 -->
<input class="h-9 sm:h-10 ...">

<!-- 文字大小响应式 -->
<div class="text-[12px] sm:text-[13px] ...">

<!-- 内边距响应式 -->
<button class="px-4 sm:px-5 ...">
```

## 11. 关键原则

1. **一致性优先**: 所有按钮高度统一使用 `h-10`，圆角统一使用 `rounded-lg`
2. **色彩语义**: 蓝色用于主要操作，绿色用于成功状态，红色用于错误状态
3. **层级分明**: 页面背景 → 卡片背景 → 输入框背景，依次加深
4. **响应式设计**: 移动端优先，小屏幕用较小尺寸，大屏幕自动适配
5. **可访问性**: 确保文字与背景有足够对比度，使用 `aria-label` 等属性
