# 3.40 开源部署 Agent（Deploy Agent）

## 概述

v3.1.1 新增「开源部署 Agent」模式，用户在 AI 对话页面提供项目名称/GitHub 地址 + 部署需求，Agent 自动完成从创建云服务器到安装部署的全流程。同版本还新增了「报错分析 Agent」和可配置的最大工具调用轮次。

## 需求背景

- 用户希望一句话完成"在云上部署 XXX"的操作，无需手动查模板、创建场景、SSH 登录安装
- 已有的 Agent 模式是通用运维助手，缺少部署导向的工作流引导
- 已有的模板生成模式只输出文本，无法自动保存和执行

## 架构设计

### 核心思路

复用已有的 `AgentChatStream` 的 agentic loop（tool calling 循环），不新建独立的 stream 实现，而是：

1. 提取 `runAgentLoop()` 通用函数，所有 Agent 类模式共享
2. 通过不同的 System Prompt 引导不同行为
3. 新增 `save_template_files` MCP 工具，让 AI 能自动保存生成的模板

### 关键流程

```
用户: "部署 nginx 在阿里云，监听 8080 端口"
  │
  ▼
Agent: 理解需求 → list_cases 查看现有场景
  │
  ├─ 有合适的运行中场景 ─────────────────────┐
  │                                           ▼
  │                         exec_command 安装 nginx
  │                         exec_command 配置 8080 端口
  │                         get_case_outputs 获取 IP
  │                         返回部署结果
  │
  └─ 没有合适场景 ──────────────────────────────┐
                                                 ▼
                          list_templates 查看本地模板
                          search_templates 搜索仓库
                               │
                               ├─ 有合适模板 → pull_template
                               └─ 没有 → save_template_files（AI 生成）
                               │
                               ▼
                          plan_case 创建场景
                          start_case 启动场景
                          等待就绪...
                          exec_command 安装软件
                          get_case_outputs 获取 IP
                          返回部署结果
```

## 关键实现细节

### 1. runAgentLoop() — 通用 Agent 循环

```go
func (a *App) runAgentLoop(
    conversationId string,
    messages []AIChatMessage,
    promptTemplate string,      // System Prompt 模板（含 %s 占位符）
    defaultMaxRounds int,       // 默认最大轮次
    timeout time.Duration,      // 上下文超时
) error
```

- `AgentChatStream` 调用：defaultMaxRounds=20, timeout=5min
- `DeployAgentChatStream` 调用：defaultMaxRounds=30, timeout=10min
- 如果用户在 AI 配置中设置了 `MaxToolRounds > 0`，优先使用用户值

### 2. DeployAgentSystemPrompt

System Prompt 包含以下关键部分：

- **角色定义**：开源项目自动部署助手
- **6 步工作流**：理解需求 → 查找现有场景 → 准备模板 → 创建启动场景 → 部署软件 → 汇报结果
- **模板生成规则**：case.json 格式、main.tf 结构、云厂商 Provider 参考、实例规格建议
- **常用软件部署命令**：nginx / docker / redis / git clone / go / python 项目安装参考
- **注意事项**：非交互模式（apt-get -y）、nohup 后台执行、plan 报错自动修正

### 3. save_template_files MCP 工具

```
工具名: save_template_files
参数:
  - template_name (string, required): 模板名，必须以 ai- 前缀开头
  - files (object, required): 文件名→内容映射（case.json, main.tf, variables.tf 等）
返回: 保存路径和文件数量
```

- 通过 `AppBridge.MCPSaveTemplateFiles()` 调用 `App.SaveTemplateFiles()`
- 自动加 `ai-` 前缀（如果没有），防止覆盖用户模板
- 只允许写入白名单后缀（.tf / .json / .tfvars / .yaml / .sh / .md 等）

### 4. 最大工具调用轮次配置

- 存储在 `AIConfig.MaxToolRounds`（`json:"maxToolRounds,omitempty"`）
- 前端：凭据管理页面 AI 配置区的 range 滑块（5-50，步进 5）
- 后端：`runAgentLoop()` 中 `if aiConfig.MaxToolRounds > 0 { maxRounds = aiConfig.MaxToolRounds }`
- 默认值：Agent 模式 20 轮，Deploy 模式 30 轮

### 5. 报错分析 Agent

- 模式 ID: `errorAnalysis`
- System Prompt: `ErrorAnalysisChatSystemPrompt`，包含 Terraform 生命周期、云厂商错误码、模板结构知识
- 自动注入模板文件上下文（通过 `gatherTemplateContext()` 读取 .tf / case.json / terraform.tfvars）
- 从 Cases.svelte / CustomDeploymentList.svelte 的"AI 分析错误原因"按钮跳转，通过 `localStorage('ai-chat-pending-error')` 传递数据并自动发送

### 6. 前端模式列表（AIChat.svelte）

```javascript
const modes = [
  { id: 'free',          labelKey: 'aiChatFreeChat',       icon: '💬' },
  { id: 'agent',         labelKey: 'aiChatAgent',          icon: '✓'  },
  { id: 'deploy',        labelKey: 'aiChatDeploy',         icon: '🚀' },
  { id: 'errorAnalysis', labelKey: 'aiChatErrorAnalysis',  icon: '⚠'  },
  { id: 'generate',      labelKey: 'aiChatGenTemplate',    icon: '</>' },
  { id: 'recommend',     labelKey: 'aiChatRecommend',      icon: '💡' },
  { id: 'cost',          labelKey: 'aiChatCostOpt',        icon: '$'  },
];
```

`sendMessage()` 路由：
- `mode === 'agent'` → `AgentChatStream()`（通用 Agent，10→20 轮）
- `mode === 'deploy'` → `DeployAgentChatStream()`（部署 Agent，30 轮）
- 其他模式 → `AIChatStream()`（非工具调用，纯流式）

## 维护注意事项

1. **新增 Agent 模式**：只需在 `prompt.go` 添加 Prompt，在 `app_ai_chat.go` 添加 wrapper 函数，在 `AIChat.svelte` 添加 mode 条目
2. **工具调用轮次不够**：用户可在凭据管理页面调整，最大 50 轮
3. **deploy prompt 调优**：修改 `DeployAgentSystemPrompt` 即可，不需要改代码逻辑
4. **新增 MCP 工具**：在 `mcp.go` 添加定义 + dispatch + 实现，Agent 会自动获取新工具
5. **Wails 绑定**：新增 Go 导出函数后需运行 `wails generate module`，如果失败需手动添加到 `App.js` / `App.d.ts`
