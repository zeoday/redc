# GUI 自动打包发布功能实现

## 概述

本文档记录 redc-gui 桌面应用通过 GitHub Actions 自动打包 macOS 和 Windows 版本的实现，供后续维护参考。

---

## 1. 工作流程文件

文件位置：`.github/workflows/release-gui.yml`

---

## 2. 触发方式

### 2.1 自动触发

当推送 `v*` 标签时自动触发：

```yaml
on:
  push:
    tags:
      - 'v*'
```

### 2.2 手动触发

支持在 GitHub 界面手动运行：

```yaml
on:
  push:
    tags:
      - 'v*'
  workflow_dispatch:
```

---

## 3. 构建任务

### 3.1 macOS 构建

| 项目 | 值 |
|------|-----|
| Runner | macos-latest |
| 平台 | darwin/universal（Intel + Apple Silicon）|
| 输出 | redc-gui-macos-universal.zip |

**关键步骤：**

```yaml
- name: Build macOS
  run: wails build -platform darwin/universal
  env:
    CGO_ENABLED: 1

- name: Package as zip
  run: |
    cd build/bin
    zip -r redc-gui-macos-universal.zip redc-gui.app
```

### 3.2 Windows 构建

| 项目 | 值 |
|------|-----|
| Runner | windows-latest |
| 平台 | windows/amd64 |
| 输出 | redc-gui-windows-amd64.zip |

**关键步骤：**

```yaml
- name: Build Windows
  run: wails build -platform windows/amd64
  env:
    CGO_ENABLED: 1

- name: Package as zip
  run: |
    cd build/bin
    Compress-Archive -Path redc-gui.exe -DestinationPath redc-gui-windows-amd64.zip
```

## 4. 依赖安装

### 4.1 Go 环境

```yaml
- name: Setup Go Environment
  uses: actions/setup-go@v5
  with:
    go-version: ${{ env.GO_VERSION }}  # 1.22.1
```

### 4.2 Node.js 环境

```yaml
- name: Setup Node.js
  uses: actions/setup-node@v4
  with:
    node-version: ${{ env.NODE_VERSION }}  # 20
```

### 4.3 Wails 安装

```yaml
- name: Install Wails
  run: go install github.com/wailsapp/wails/v2/cmd/wails@latest
```

### 4.4 前端依赖

注意：前端代码在 `frontend/` 目录下，需要先 cd 进入：

```yaml
- name: Install frontend dependencies
  run: |
    cd frontend
    npm install
```

## 5. 缓存优化

### 5.1 macOS 缓存

```yaml
- name: Cache npm dependencies
  uses: actions/cache@v4
  with:
    path: ${{ steps.npm-cache-dir.outputs.dir }}
    key: ${{ runner.os }}-npm-${{ hashFiles('frontend/package-lock.json') }}
```

### 5.2 Windows 缓存

```yaml
- name: Cache npm dependencies
  uses: actions/cache@v4
  with:
    path: |
      ~\AppData\Local\npm-cache
      ~\AppData\Local\go-build
    key: ${{ runner.os }}-npm-${{ hashFiles('frontend/package-lock.json') }}
```

## 6. 发布流程

### 6.1 收集产物

```yaml
- name: Download macOS Artifact
  uses: actions/download-artifact@v4
  with:
    name: redc-gui-macos
    path: dist

- name: Download Windows Artifact
  uses: actions/download-artifact@v4
  with:
    name: redc-gui-windows
    path: dist
```

### 6.2 创建 Release

```yaml
- name: Release
  uses: softprops/action-gh-release@v2
  with:
    files: dist/*
    generate_release_notes: true
  env:
    GITHUB_TOKEN: ${{ secrets.RELEASE_GH_TOKEN }}
```

## 13. 注意事项

### 13.1 CGO

macOS 和 Windows 构建需要启用 CGO：
```yaml
CGO_ENABLED: 1
```

### 13.2 前端目录

redc 项目是前后端分离结构：
- 源码在根目录
- 前端代码在 `frontend/` 子目录
- 构建命令需要正确切换目录

### 13.3 Wails 版本

使用最新的 Wails v2：
```bash
go install github.com/wailsapp/wails/v2/cmd/wails@latest
```

### 13.4 Universal vs Intel/Apple Silicon

macOS 构建使用 `darwin/universal`，生成的应用同时支持 Intel 和 Apple Silicon 芯片。

如果需要分别构建：
```bash
# Intel
wails build -platform darwin

# Apple Silicon
wails build -platform darwin/arm64
```
