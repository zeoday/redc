# 云厂商账单查询功能实现

## 概述

本文档记录在 redc 和 redc-gui 中添加云厂商账单（当月费用）查询功能的过程，供后续维护参考。

---

## 1. 功能设计

### 1.1 需求分析

仪表盘需要显示各云厂商的当月累计费用（账单），与"账户余额"（当前账户可用余额）不同。

| 类型 | 说明 |
|------|------|
| 账户余额 | 当前账户可用余额/信用额度 |
| 当月账单 | 当月累计消费金额 |

### 1.2 支持的云厂商

| 云厂商 | API | 说明 |
|--------|-----|------|
| AWS | Cost Explorer API | 查询当月 UnblendedCost |
| Vultr | Billing History API | 查询当月消费记录 |

## 2. 后端实现 (mod/)

### 2.1 AWS 账单查询

文件: `mod/balance_aws.go`

```go
func QueryAWSBill(accessKey string, secretKey string, region string) (string, string, error)
```

实现要点:
- 使用 AWS SDK 的 Cost Explorer API
- `Granularity: MONTHLY` - 按月查询
- `Metrics: ["UnblendedCost", "AmortizedCost"]` - 获取未分摊成本
- 返回格式: `("金额", "USD")`

### 2.2 Vultr 账单查询

文件: `mod/balance_vultr.go`

```go
func QueryVultrBill(apiKey string) (string, string, error)
```

实现要点:
- 使用 Vultr API v2 `GET /v2/billing/history`
- 按时间范围筛选当月记录
- 只计算消耗类费用 (instance, bandwidth, snapshot)，排除充值
- 返回格式: `("金额", "USD")`

## 3. GUI 后端 (app.go)

### 3.1 数据结构

文件: `app.go`

```go
type BillInfo struct {
    Provider    string `json:"provider"`
    Month       string `json:"month"`
    TotalAmount string `json:"totalAmount"`
    Currency    string `json:"currency"`
    StartDate   string `json:"startDate"`
    EndDate     string `json:"endDate"`
    Error       string `json:"error"`
}
```

### 3.2 获取账单函数

```go
func (a *App) GetBills(providers []string) ([]BillInfo, error)
```

支持参数:
- `providers`: 要查询的云厂商列表，默认 `["aws", "vultr"]`

实现逻辑:
1. 读取配置文件获取凭据
2. 遍历各云厂商调用对应的账单查询函数
3. 返回账单信息列表

### 3.3 在 GetBalances 中添加 AWS

文件: `app.go` - `GetBalances` 函数

```go
case "aws":
    amount, currency, err := redc.QueryAWSBill(conf.Providers.Aws.AccessKey, conf.Providers.Aws.SecretKey, conf.Providers.Aws.Region)
    // ...
```

注意: 账户余额中仍然保留 AWS（从之前版本就有），但实际上 AWS 没有余额 API，所以显示的是账单而非余额。

## 6. 注意事项

### 6.1 AWS Cost Explorer

1. **需要启用 Cost Explorer**: 大多数 AWS 账户默认已启用
2. **IAM 权限**: 需要 `ce:GetCostAndUsage` 权限
3. ** granularity 区别**:
   - `MONTHLY`: 返回当月预估（包含整月，即使还没到月底）
   - `DAILY`: 返回每天明细，需要累加
4. **费用类型**:
   - `UnblendedCost`: 未分摊成本
   - `AmortizedCost`: 分摊成本（适合预留实例）

### 6.2 Vultr API

1. **API 端点**: `GET /v2/billing/history`
2. **时间筛选**: 使用 `start_date` 和 `end_date` 参数
3. **费用类别**: 需要过滤掉充值类型的记录

### 6.3 GCP 账单

GCP 账单 API 需要完整的 OAuth2 认证流程，暂时未实现。如需添加:
- 需要使用 `google.golang.org/api/cloudbilling/v1` 库
- 需要通过服务账号 JWT 获取访问令牌
- 调用 `https://cloudbilling.googleapis.com/v1/billingAccounts/{account}/costs`
