# GUI i18n and Welcome Dialog Design

## Overview

This document describes the internationalization (i18n) system and welcome dialog implementation for RedC GUI.

---

## 1. Internationalization (i18n) System

### 1.1 Implementation Overview

The GUI supports both Chinese and English languages. Language settings are persisted in the GUI configuration file.

### 1.2 Configuration Storage

**File**: `~/redc/gui_settings.json`

```json
{
  "language": "zh",
  "welcomeDialogShown": ""
}
```

| Field | Type | Description |
|-------|------|-------------|
| language | string | UI language: "zh" (Chinese) or "en" (English) |
| welcomeDialogShown | string | Welcome dialog status: "" (first time), "hidden" (user opted out) |

### 1.3 Backend Implementation (Go)

**Files Modified**:
- `mod/config.go` - GUISettings struct
- `app.go` - GetLanguage, SetLanguage, GetShowWelcomeDialog, SetShowWelcomeDialog functions

#### Language Detection

```go
// GetLanguage returns the current GUI language
func (a *App) GetLanguage() string {
    settings, err := redc.LoadGUISettings()
    if err != nil {
        return detectSystemLanguage() // First time: detect system language
    }
    if settings.Language == "" {
        return detectSystemLanguage() // No saved language: detect system language
    }
    return settings.Language // Return saved language
}
```

#### System Language Detection

```go
func detectSystemLanguage() string {
    lang := getSystemLocale()
    if len(lang) >= 2 && lang[:2] == "zh" {
        return "zh"
    }
    return "en"
}

func getSystemLocale() string {
    // Check LC_ALL, LC_MESSAGES, LANG, LANGUAGE environment variables
    locales := []string{"LC_ALL", "LC_MESSAGES", "LANG", "LANGUAGE"}
    for _, env := range locales {
        if val := os.Getenv(env); val != "" {
            parts := strings.Split(val, ".")
            if len(parts) > 0 {
                return strings.ToLower(parts[0])
            }
        }
    }
    return "en"
}
```

**Logic**:
1. First launch: Detect system language automatically
2. User switches language: Save to `gui_settings.json`
3. Subsequent launches: Use saved language (not detected again)

## 2. Welcome Dialog

### 2.1 Implementation Overview

A welcome dialog is shown on first launch to introduce the application features. Users can opt out of showing it again.

### 2.2 Configuration Storage

**Field**: `welcomeDialogShown` in `gui_settings.json`

| Value | Meaning | Show Dialog? |
|-------|---------|---------------|
| "" (empty) | First time using | ✅ Yes |
| "hidden" | User clicked "Don't show again" | ❌ No |

### 2.3 Backend Implementation (Go)

```go
// SetShowWelcomeDialog saves user's choice
func (a *App) SetShowWelcomeDialog(shown bool) error {
    settings, err := redc.LoadGUISettings()
    if err != nil {
        return err
    }
    if shown {
        settings.WelcomeDialogShown = "true"
    } else {
        settings.WelcomeDialogShown = "hidden"
    }
    return redc.SaveGUISettings(settings)
}

// GetShowWelcomeDialog checks if should show dialog
func (a *App) GetShowWelcomeDialog() bool {
    settings, err := redc.LoadGUISettings()
    if err != nil {
        return true // First time, show
    }
    return settings.WelcomeDialogShown == "" // Show only if empty
}
```

### 2.4 Frontend Implementation (Svelte)

**Files Createdfrontend/src/components/W**:
- `elcome/WelcomeDialog.svelte`

#### Avoiding Flash

To prevent the dialog from flashing on screen, the frontend uses a two-step approach:

1. Initial state: `welcomeDialogReady = false` (dialog hidden)
2. After data loads: Call `GetShowWelcomeDialog()` to check
3. If should show: Set `welcomeDialogReady = true` (dialog appears)

```svelte
// App.svelte
let welcomeDialogReady = $state(false);

// After data loads
const shouldShowWelcome = await GetShowWelcomeDialog();
if (shouldShowWelcome) {
    welcomeDialogReady = true;
}
```

#### Component Usage

```svelte
<WelcomeDialog 
    {t} 
    show={welcomeDialogReady} 
    onClose={() => welcomeDialogReady = false} 
/>
```

## 3. AI Language Follow-up

### 3.1 Overview

AI responses follow the UI language setting for better user experience.

### 3.2 Implementation

In all AI-related functions, the system prompt includes language instruction:

```go
// Get UI language
uiLang := a.GetLanguage()
langPrompt := "请用中文回复" // Chinese
if uiLang == "en" {
    langPrompt = "Please reply in English" // English
}

// Add to system prompt
systemPrompt := existingPrompt + "\n" + langPrompt
```

## 4. Key Design Decisions

### 4.1 Language Persistence

- **Why**: Users should not need to switch language every time they open the app
- **How**: Save to `gui_settings.json` via Go backend

### 4.2 System Language Detection

- **Why**: Provide good out-of-box experience for first-time users
- **When**: Only on first launch (when `language` field is empty)
- **How**: Check environment variables (LC_ALL, LC_MESSAGES, LANG)

### 4.3 Welcome Dialog Avoid Flash

- **Why**: Initial `true` value caused visible flash when overridden by backend `false`
- **How**: Use `welcomeDialogReady = false` initially, set to `true` only after checking backend

### 4.4 String Instead of Bool for WelcomeDialogShown

- **Why**: Go's bool cannot distinguish "not set" from "set to false"
- **How**: Use string: "" (first time), "hidden" (opted out)

## 5. Future Maintenance Notes

### 5.1 Adding New Translations

1. Add key to both Chinese and English sections in `i18n.js`
2. Always provide fallback: `t.key || 'default'`

### 5.2 Adding New GUI Settings

1. Add field to `GUISettings` struct in `mod/config.go`
2. Handle default value in `LoadGUISettings()` for new installs
3. Add getter/setter functions in `app.go` if needed

### 5.3 Changing Default Language

To change the default language from Chinese to English:
- Modify `detectSystemLanguage()` in `app.go`
- Or modify the default in `LoadGUISettings()` in `mod/config.go`
