# 代理配置功能实现

## 概述

本文档记录 redc-gui 代理配置功能的设计与实现，供后续维护参考。

---

## 1. 功能需求

GUI 界面配置代理后，需要让以下组件都能使用代理：
1. Terraform 子进程
2. AI API 调用
3. 云厂商余额/账单查询（Vultr、GCP 等）
4. Cloudflare API 调用

---

## 2. 架构设计

### 2.1 配置存储

代理配置保存在 `~/redc/gui_settings.json` 文件中：

```json
{
  "disableRightClick": true,
  "notificationEnabled": false,
  "debugEnabled": false,
  "httpProxy": "http://127.0.0.1:7890",
  "httpsProxy": "http://127.0.0.1:7890",
  "socks5Proxy": "",
  "noProxy": "localhost,127.0.0.1"
}
```

### 2.2 核心函数

在 `mod/config.go` 中提供两个核心函数：

```go
// 获取代理配置（优先 GUI 设置，回退环境变量）
func GetProxySettings() (httpProxy, httpsProxy, noProxy string)

// 获取单个代理 URL（优先 httpsProxy）
func GetProxyURL() string
```

### 2.3 HTTP Client 工厂函数

```go
// 创建使用代理的 HTTP Client
func NewProxyHTTPClient(timeout time.Duration) *http.Client
```

## 3. 组件改造

### 3.1 AI Client

**文件**: `mod/ai/client.go`

```go
func NewClient(provider, apiKey, baseURL, model string) *Client {
    proxyURL := mod.GetProxyURL()

    var transport *http.Transport
    if proxyURL != "" {
        if proxy, err := url.Parse(proxyURL); err == nil {
            transport = &http.Transport{
                Proxy: http.ProxyURL(proxy),
                // ... 其他配置
            }
        }
    }

    // 如果没有配置代理，使用默认 Transport
    if transport == nil {
        transport = &http.Transport{...}
    }

    return &Client{
        client: &http.Client{
            Transport: transport,
        },
    }
}
```

**要点**:
- 显式设置 `Proxy` 属性
- 保留原有的连接超时配置
- 没有代理时使用默认 Transport

## 4. 代理配置流程

### 4.1 保存代理配置

1. 用户在设置页面填写代理地址
2. 前端调用 `SaveProxyConfig(httpProxy, httpsProxy, noProxy)`
3. 后端设置当前进程环境变量
4. 后端保存到 `gui_settings.json`

```go
func (a *App) SaveProxyConfig(httpProxy, httpsProxy, noProxy string) error {
    // 设置环境变量
    os.Setenv("HTTP_PROXY", httpProxy)
    // ...

    // 持久化保存
    settings, _ := redc.LoadGUISettings()
    settings.HttpProxy = httpProxy
    settings.HttpsProxy = httpsProxy
    settings.NoProxy = noProxy
    redc.SaveGUISettings(settings)
}
```

### 4.2 应用启动时加载代理

**文件**: `app.go` - `startup` 函数

```go
func (a *App) startup(ctx context.Context) {
    // 加载 GUI 设置
    if settings, err := redc.LoadGUISettings(); err == nil {
        // 应用代理设置到环境变量
        if settings.HttpProxy != "" {
            os.Setenv("HTTP_PROXY", settings.HttpProxy)
        }
        // ...
    }
}
```

### 4.3 HTTP Client 请求时使用代理

```go
func NewProxyHTTPClient(timeout time.Duration) *http.Client {
    proxyURL := GetProxyURL()

    if proxyURL != "" {
        if proxy, err := url.Parse(proxyURL); err == nil {
            return &http.Client{
                Timeout: timeout,
                Transport: &http.Transport{
                    Proxy: http.ProxyURL(proxy),
                },
            }
        }
    }

    return &http.Client{Timeout: timeout}
}
```

## 5. 代理优先级

```
SOCKS5_PROXY > HTTPS_PROXY > HTTP_PROXY > 环境变量
```

在 `GetProxyURL()` 中：
1. 首先检查 `socks5Proxy`（最优先）
2. 然后检查 `httpsProxy`
3. 然后检查 `httpProxy`
4. 如果都为空，返回空字符串（不走代理）

注意：SOCKS5 代理使用 `ALL_PROXY` 环境变量传递给需要代理的组件。

## 7. 注意事项

### 7.1 AI Client 特殊处理

AI Client 使用自定义 `Transport` 而非默认的 HTTP Transport，原代码：
```go
client := &http.Client{
    Transport: &http.Transport{
        DialContext: (&net.Dialer{...}).DialContext,  // 直接 Dial，不走代理！
    },
}
```

修改后：
```go
transport := &http.Transport{
    Proxy: http.ProxyURL(proxy),  // 显式设置代理
    DialContext: ...,
}
```

### 7.2 Terraform 子进程

Terraform 通过环境变量获取代理，而不是通过代码传递。需要：
1. 在启动时设置环境变量
2. 确保 `TF_CLI_CONFIG_FILE` 等配置也通过环境变量传递

### 7.3 代理 URL 格式

支持的代理 URL 格式：
- `http://proxy:port`
- `https://proxy:port`
- `socks5://proxy:port`

### 7.4 默认值

默认值的默认值：
```go
LoadedGUISettings = &GUISettings{
    DisableRightClick:   true,
    NotificationEnabled: false,
    DebugEnabled:        false,
    HttpProxy:          "",
    HttpsProxy:         "",
    Socks5Proxy:        "",
    NoProxy:            "",
}
```

| 字段 | 默认值 | 说明 |
|------|--------|------|
| httpProxy | "" | 空字符串表示不使用 |
| httpsProxy | "" | 空字符串表示不使用 |
| socks5Proxy | "" | 空字符串表示不使用 |
| noProxy | "" | 空格分割的主机名列表 |
