# CTYun (天翼云) 云厂商支持实现

## 概述

本文档记录在 redc 和 redc-gui 中添加 CTYun (天翼云) 云厂商支持的过程，供后续维护参考。

### 1.1 添加 CTYun 配置结构体

文件: `mod/config.go`

```go
Ctyun struct {
    AccessKey string `yaml:"CTYUN_AK" env:"CTYUN_AK"`
    SecretKey string `yaml:"CTYUN_SK" env:"CTYUN_SK"`

```

## 3. 密码自动生成 (mod/case.go)

### 3.1 添加 CTYun 密码生成支持

文件: `mod/case.go`

在 `ensureProviderVars` 函数中添加:

```go
// 处理天翼云 CTYun
if provider == "ctyun" {
    // 如果模板需要 access_key 且用户未提供，则从配置中获取
    if (vars["access_key"] == "" || isPlaceholderSecret(vars["access_key"])) && conf.Providers.Ctyun.AccessKey != "" {
        vars["access_key"] = conf.Providers.Ctyun.AccessKey
    }
    if (vars["secret_key"] == "" || isPlaceholderSecret(vars["secret_key"])) && conf.Providers.Ctyun.SecretKey != "" {
        vars["secret_key"] = conf.Providers.Ctyun.SecretKey
    }
    // 如果模板需要 instance_password 且用户未提供，则自动生成
    if vars["instance_password"] == "" {
        vars["instance_password"] = generateInstancePassword()
        gologger.Info().Msgf("已自动生成天翼云实例密码")
    }
}
```

## 4. GUI 凭据页面 (app.go)

### 4.1 获取凭据配置

文件: `app.go`

在 `GetProvidersConfig` 函数中添加 CTYun 凭据字段:

```go
{
    Name: "Ctyun",
    Fields: map[string]string{
        "accessKey": maskValue(conf.Providers.Ctyun.AccessKey),
        "secretKey": maskValue(conf.Providers.Ctyun.SecretKey),
        "region":    conf.Providers.Ctyun.Region,
    },
    HasSecrets: map[string]bool{
        "accessKey": conf.Providers.Ctyun.AccessKey != "",
        "secretKey": conf.Providers.Ctyun.SecretKey != "",
    },
},
```

### 4.2 保存凭据配置

文件: `app.go`

在 `SaveProvidersConfig` 函数中添加:

```go
case "Ctyun":
    if v, ok := fields["accessKey"]; ok && v != "" {
        conf.Providers.Ctyun.AccessKey = v
    }
    if v, ok := fields["secretKey"]; ok && v != "" {
        conf.Providers.Ctyun.SecretKey = v
    }
    if v, ok := fields["region"]; ok {
        conf.Providers.Ctyun.Region = v
    }
```

## 8. 注意事项

1. **CTYun Provider 环境变量**: 使用 `CTYUN_AK` 和 `CTYUN_SK` (不是 ACCESS_KEY)
2. **可用区**: 必须指定 `az_name` 参数，不能为空
3. **镜像和规格**: 建议使用 data source 查询，不要硬编码 ID
4. **资源命名**: 使用 `random_id` 生成随机后缀，避免重复创建冲突
5. **安全组规则**: CTYun Terraform provider 存在 bug，建议不使用安全组规则
6. **凭据传递**: 通过环境变量 `CTYUN_AK` 和 `CTYUN_SK` 传递给 Terraform provider
