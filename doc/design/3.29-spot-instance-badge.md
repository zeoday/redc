# 抢占式实例标签（Spot Instance Badge）设计文档

## 1. 背景

RedC 支持多个云厂商的抢占式/竞价实例（Spot Instance），但 GUI 中只有自定义部署列表通过 `config.is_spot_instance` 字段显示了抢占式标签。预定义场景页面和资源拓扑预览页面缺少此标识，用户无法直观判断某个场景是否使用了抢占式实例。

## 2. 各云厂商 Spot 标识方式

不同云厂商在 Terraform 中表示抢占式实例的方式不同：

| 云厂商 | Terraform 字段 | 抢占式值 | 示例 |
|--------|---------------|----------|------|
| 阿里云 | `spot_strategy` | `"SpotWithPriceLimit"`, `"SpotAsPriceGo"` | `spot_strategy = "SpotWithPriceLimit"` |
| 火山引擎 | `spot_strategy` (由 `is_spot_instance` 变量控制) | `"SpotAsPriceGo"` | `spot_strategy = var.is_spot_instance ? "SpotAsPriceGo" : "NoSpot"` |
| AWS | `instance_market_options.market_type` | `"spot"` | `market_type = "spot"` |

### 2.1 关键发现

预定义场景的 Spot 信息有两种来源：
- **变量方式**（base-templates）：通过 `is_spot_instance` 变量控制，值在 `Case.Parameter` 中
- **硬编码方式**（如 `aliyun/proxy`）：直接在 `main.tf` 写死 `spot_strategy = "SpotWithPriceLimit"`，`Parameter` 中无 spot 信息

因此不能仅依赖参数检测，需要扫描 `.tf` 文件。

## 3. 检测方案

### 3.1 两层检测架构

```
┌─────────────────────────────────────────────────────────┐
│  场景列表（Cases / CustomDeploymentList）                 │
│                                                          │
│  预定义场景:  detectSpotFromTfFiles(case.Path)            │
│              → 扫描 .tf 文件中的 spot 关键字               │
│              → CaseInfo.IsSpotInstance                    │
│                                                          │
│  自定义部署:  deployment.config.is_spot_instance           │
│              → 已有字段，直接使用                           │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│  资源拓扑预览 Modal                                       │
│                                                          │
│  detectSpotInstance(rc.Change.After)                      │
│  → 从 terraform plan JSON 的 after-values 中检测          │
│  → PlanPreview.IsSpotInstance                             │
└─────────────────────────────────────────────────────────┘
```

### 3.2 `detectSpotFromTfFiles()` — 场景列表检测

**位置**: `app_scene.go`

**逻辑**:
1. 扫描 `case.Path` 下所有 `*.tf` 文件
2. 搜索关键词 `spot_strategy`、`market_type`、`is_spot_instance`
3. 跳过注释行（`#` 或 `//` 开头）
4. 按关键词分别判断：
   - `spot_strategy`: 包含 `"Spot` 且不是 `"NoSpot"` → 是抢占式
   - `market_type`: 包含 `"spot"` → 是抢占式
   - `is_spot_instance`: 包含 `= true` → 是抢占式

**性能**: 仅读取 `.tf` 文本文件（通常几 KB），在 `ListCases()` 调用时执行，开销极小。

### 3.3 `detectSpotInstance()` — 拓扑预览检测

**位置**: `app_scene.go`

**逻辑**:
1. 从 `terraform show -json` 解析出的 plan 数据中，检查每个资源的 after-values
2. 检查字段：
   - `spot_strategy`: 非空且非 `"NoSpot"` → 是抢占式
   - `instance_market_options[].market_type`: 值为 `"spot"` → 是抢占式
3. 任一资源匹配即标记 `PlanPreview.IsSpotInstance = true`

## 4. 数据流

### 4.1 预定义场景列表

```
ListCases()
  → LoadProjectCases()
  → 遍历 cases
    → detectSpotFromTfFiles(c.Path)
    → CaseInfo { ..., IsSpotInstance: true/false }
  → 前端 Cases.svelte
    → {#if c.isSpotInstance} <badge> {/if}
```

### 4.2 自定义部署列表

```
已有实现（无变更）:
deployment.config.is_spot_instance
  → {#if deployment.config?.is_spot_instance} <badge> {/if}
```

### 4.3 资源拓扑预览

```
GetCasePlanPreview() / GetDeploymentPlanPreview()
  → buildPlanPreview()
    → 遍历 resourceChanges
      → detectSpotInstance(rc.Change.After)
      → PlanPreview { ..., IsSpotInstance: true/false }
  → 前端 modal header
    → {#if planPreviewModal.data?.isSpotInstance} <badge> {/if}
```

| 文件 | 修改内容 |

## 6. 标签样式

所有位置使用统一的琥珀色标签样式（与自定义部署列表一致）：

```html
<span class="px-1.5 py-0.5 text-[10px] font-medium text-amber-700 bg-amber-50 border border-amber-200 rounded">
  抢占式
</span>
```

i18n key: `spotInstance`（中文 "抢占式"，英文 "Spot Instance"）

## 7. 扩展新云厂商

如果未来支持新的云厂商抢占式实例，只需在两个函数中添加检测逻辑：

1. `detectSpotFromTfFiles()` — 添加新的关键词和匹配规则
2. `detectSpotInstance()` — 添加新的 plan after-values 字段检查

无需修改前端代码。
